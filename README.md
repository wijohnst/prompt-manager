# pm — prompt-manager

Compose AI agent prompts from modular markdown files. `pm build` assembles a prompt for a named agent by combining its base, directives, resources, roles, vocations, and skills into a single output file.

## Requirements

- Node.js >= 22

## Installation

```sh
npm install -g pm
```

## Quickstart

```sh
pm build <agent>
```

Reads configuration from `pm.toml` in the current directory. Writes the assembled prompt to `.pm/build/<agent>.md`.

**Example:**

```sh
pm build founder
# built: /path/to/project/.pm/build/founder.md
```

## Project layout

```
pm.toml                        # build configuration (see below)
.pm/
  prompts/
    base/                      # one file per agent, defines role and mission
    directives/                # injected into every agent automatically
    skills/                    # capability modules included per agent or role
    vocations/                 # framing layers applied before skills
    workflows/                 # shared workflow definitions
  build/                       # output — generated by pm build, one file per agent
  docs/                        # shared reference data (e.g. staff directory)
```

## pm.toml

`pm.toml` is the build configuration file. It lives at the project root.

### `[build]`

```toml
[build]
output = ".pm/build"
```

`output` — directory where built prompts are written.

### `[[build.directives]]`

Directives are injected into every agent's built prompt automatically. No per-agent declaration needed.

```toml
[[build.directives]]
name = "company-north-stars"
path = ".pm/prompts/directives/company-north-stars.md"
include = "inline"
```

### `[[build.resources]]`

Shared reference data injected into every agent's built prompt.

```toml
[[build.resources]]
name = "staff-directory"
path = ".pm/docs/org/staff-directory.md"
include = "inline"
```

### `[roles.<name>]`

A role is a named set of skills assigned to one or more agents.

```toml
[roles.org-agent]

[[roles.org-agent.skills]]
name = "mail"
path = ".pm/prompts/skills/mail.md"
include = "inline"
```

### `[agents.<name>]`

Each agent entry defines what goes into its built prompt.

```toml
[agents.founder]
base = ".pm/prompts/base/founder.md"    # required — the agent's base prompt
platform = "claude-code"                # platform hint written to output frontmatter
roles = ["org-agent"]                   # skills from these roles are included
```

**Vocations** — composed before skills:

```toml
[[agents.founder.vocations]]
name = "delegator"
path = ".pm/prompts/vocations/delegator.md"
include = "inline"
```

**Skills** — per-agent capability modules:

```toml
[[agents.founder.skills]]
name = "prompt-iteration"
path = ".pm/prompts/skills/prompt-iteration.md"
include = "inline"
```

`include` accepts `"inline"` (content embedded) or `"by-reference"` (path referenced but not embedded).

**Memory** — agent-specific memory files appended to the prompt:

```toml
[agents.founder.memory]
path = ".pm/memory"
files = ["feedback_no_internal_memory.md", "project_current-state.md"]
```

## Output format

Built prompts are markdown files with a TOML frontmatter block:

```
---
[agent]
name = "founder"
platform = "claude-code"

[[skills]]
name = "mail"
include = "inline"
path = ".pm/prompts/skills/mail.md"
---

<!-- directive: company-north-stars -->
...inlined directive content...

<!-- base prompt content -->
...
```

## Usage

```
pm build <agent>     compose and build a prompt for the named agent
pm --version         print version
pm build --help      show help for the build command
```
